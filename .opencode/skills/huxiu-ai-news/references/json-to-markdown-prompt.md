# JSON 到 Markdown 提示词模板

## 使用方式

把 `scripts/fetch_huxiu_ai_news.py --format json --stdout` 的输出作为输入，直接喂给执行 skill 的大模型。

## 提示词

```text
你将收到一份虎嗅 AI 科技新闻的 JSON 数据。请严格把它转换成 Markdown 报告，要求如下：

1. 只使用 JSON 中提供的信息，不要编造事实。
2. 报告顶部必须输出：
   - `# 虎嗅 AI 科技热点 | YYYY-MM-DD`
   - `> 更新时间: YYYY-MM-DD HH:MM:SS`
   - `> 来源: 虎嗅 AI News`
   - `> 仓库: ai-news-daily`
   - `---`
   - `## Top N 虎嗅 AI 热点`
3. 每条内容必须输出成下面的结构：
---
## [Top N] [标题: 提炼核心价值]
> [!abstract] 核心速递
> **分类**: #category | **作者**: author_handle | **热度分**: heat_score
> **一句话总结**: summary (新手友好)
> **关键词**: #keyword1 #keyword2 #keyword3
>
> **为何值得关注**: why_it_matters (从技术深度或产品影响度分析)
>
> **互动指标**: 来源数 source_count | 收藏数 favorite_num | 评论数 comment_num
>
> **原文链接**: [虎嗅原文](url) | **发布时间**: posted_at

4. 一句话总结要求：
   - 普通新闻保持完整自然，优先用一句通俗中文概括，建议 60-100 个汉字。
   - 如果是 #模型 且属于新品发布/版本上线/模型发布，建议 90-140 个汉字，补充 1-2 个关键指标，例如参数量、上下文长度、精度、benchmark、推理速度、延迟或成本。
   - 优先保留完整子句，不要在句子中间硬截断。
   - 只有在原文信息本身缺失或明显冗长时，才使用省略号收尾。
   - 避免重复标题原文，尽量写成“前半句 + 关键指标”的自然表达。

5. 为何值得关注要求：
   - 从技术深度、产品影响、商业化价值或行业竞争角度说明。
   - 尽量具体，不要空泛。

6. 关键词要求：
   - 输出 3 个短关键词。
   - 优先使用标题和摘要中的实体名、产品名、模型名或行业关键词。

7. 排版要求：
   - 保持 Top N 从 1 开始顺序。
   - 不要添加 JSON 中没有的字段。
   - 不要输出额外解释，只输出最终 Markdown。

8. 结尾追加：
   - `*由 huxiu-ai-news Skill 自动生成*`
```

## 备注

- 这个模板适合在 skill 执行阶段作为“最后一步”的整理提示词。
- 如果只需要落盘，不需要模型二次整理，可以直接用脚本的 `markdown` 模式。
